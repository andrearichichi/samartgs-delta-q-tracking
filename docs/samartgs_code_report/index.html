Direct vs MLP nella pipeline articolata SAM-ARTGS

Versione compatta del report: cosa resta condiviso, dove nasce il branch, e cosa cambia tra direct_delta_q e mlp_q.

frozen 3DGS joint metadata articulation + render + loss direct_delta_q mlp_q
Messaggio principale Same shared pipeline, different motion parameterization.

1. Overview breve

Il tracking parte da un Gaussian model già addestrato e lo congela. Durante l'ottimizzazione non cambiano appearance o geometria del 3DGS: cambia solo la posa articolata dei Gaussian mobili.

Pipeline condivisa

  • carica RGB, mask e camera
  • carica Gaussian PLY arricchito
  • legge joint axis / pivot / part id
  • applica articolazione, renderizza, calcola loss RGB mascherata

Dove cambia

  • Direct ottimizza direttamente Δq
  • MLP ottimizza i pesi di MotionMLP
  • in entrambi i casi il renderer vede un DeformedGaussian

2. Pipeline condivisa

Tutto ciò che segue è comune ai due rami: modello Gaussian congelato, metadata articolata, trasformazione cinematica, render differenziabile e masked_rgb_loss.

4. Direct optimization

Idea

Direct è il caso più semplice: per ogni coppia di frame crea un numero Δq, lo ottimizza sul render target, e poi lo somma a q_ref.

In parole semplici È come girare una manopola numerica finché il render del Gaussian deformato coincide con l'immagine.

Δq trainabile

5. MLP optimization

Idea

MLP non crea un Δq libero. Crea una funzione q(t); valuta due tempi consecutivi e ricava Δq = q(t+1) - q(t).

In parole semplici Invece di una manopola per frame, MLP disegna una curva temporale. Il movimento è la differenza tra due punti della curva.

6. Confronto Direct vs MLP

Direct

  • ottimizza un numero: Δq
  • gradiente verso delta_q
  • commit: q_ref += committed_delta_q

MLP

  • ottimizza una funzione: q(t)
  • gradiente verso i pesi MLP
  • commit: q_ref = q_ref_committed
AspettoDirectMLP

Input motionq_reft_source, t_target
Produce motionq_ref + Δqq1 - q0

Rischiodriftoversmoothing

Esempio Direct

  • q_ref = 0.20
  • delta_q = 0.03
  • q_next = 0.20 + 0.03 = 0.23

Esempio MLP

  • q(t) = 0.20
  • q(t+1) = 0.23
  • delta_q = 0.23 - 0.20 = 0.03
In parole semplici Il risultato può essere identico; cambia dove vive l'ottimizzazione: nel numero 0.03 oppure nei pesi che producono la curva.

8. Come eseguire

python scripts/delta_q_tracking/run_sequence.py \
  --manifest configs/delta_q_tracking/dataset_manifest.json \
  --object-id USB_100109 \
  --camera-id cam_000 \
  --motion-param direct_delta_q \
  --start-frame 0 \
  --end-frame 59 \
  --num-iters 600 \
  --output-subdir usb_direct
python scripts/delta_q_tracking/run_sequence.py \
  --manifest configs/delta_q_tracking/dataset_manifest.json \
  --object-id USB_100109 \
  --camera-id cam_000 \
  --motion-param mlp_q \
  --mlp-smoothness-weight 0.001 \
  --mlp-acceleration-weight 0.001 \
  --start-frame 0 \
  --end-frame 59 \
  --num-iters 600 \
  --output-subdir usb_mlp
